# Классификация изображений (k-NN)

Учебный проект на Scala: классификация рукописных цифр из PGM-файлов методом **k ближайших соседей** (расстояние Хэмминга по полному содержимому файла изображения).

## Требования

- **JDK** 11 или новее (проверено на JDK 21)
- **sbt** 1.9+ ([установка](https://www.scala-sbt.org/download.html))

## Структура данных

| Путь | Описание |
|------|----------|
| `data/train/*.pgm` | Обучающие изображения (формат P2, 28×28) |
| `data/train/labels.txt` | По одной метке класса (0–9) на строку; порядок соответствует **отсортированным по имени** файлам `image_*.pgm` |
| `data/test/*.pgm` | Тестовые изображения |
| `data/test/labels.txt` | Создаётся при запуске — предсказанные классы |
| `data/train_indices_kfold.txt`, `data/test_indices_kfold.txt` | Индексы обучающей выборки и фолдов при кросс-валидации |

Запускать нужно из **корня репозитория**, чтобы `data/` находился в `user.dir`.

## Сборка и запуск

```bash
sbt compile
sbt run
```

Точка входа: `com.apps.classifier.ClassifierService`.

## Алгоритм

1. Загрузка обучающей выборки и теста.
2. **Кросс-валидация** (по умолчанию 5 фолдов): для каждого кандидата `K ∈ {1, 3, 5, 7, 9, 11, 15}` считается средняя доля верных ответов по фолдам; выбирается `K` с максимальной средней точностью, при равенстве — с меньшей дисперсией по фолдам.
3. Обучение на полной обучающей выборке с выбранным `K` и запись предсказаний в `data/test/labels.txt`.

Расстояние между изображениями — число позиций, в которых отличаются массивы символов файлов (с учётом заголовка PGM).

## Исправления и улучшения (ревью)

- **Зависимости**: удалена неиспользуемая `spark-mllib`; Scala и sbt приведены к актуальным версиям (совместимость артефактов `spark_*` / `scala` больше не ломает сборку).
- **k-NN**: гиперпараметр «число соседей» отделён от числа фолдов; раньше в классификатор передавалось число фолдов (10…2) как `K` соседей.
- **Обучение на фолде**: в обучающую выборку для CV ошибочно попадал текущий тестовый фолд (`cTest ++ c`); используется только индексы вне тестового фолда.
- **Голосование**: мажоритарный выбор по **меткам** соседей; раньше считалась «частота» пар `(расстояние, метка)`, что почти всегда давало некорректное голосование.
- **Данные**: изображения сортируются по имени; метки читаются построчно; проверка согласованности числа файлов и меток.
- **Пути**: каталог `data` через `user.dir` и NIO, без обрезки `getAbsolutePath`.
- **Ресурсы**: закрытие `Source` и `PrintWriter` после чтения/записи.
